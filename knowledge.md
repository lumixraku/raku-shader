## 前置条件
首先你需要安装OptiFine模组，这是运行自定义shader的必要条件。OptiFine为Minecraft添加了shader支持和更多图形选项。

## Shader包的基本结构
一个标准的shader包通常包含以下文件结构：
```
shaderpack名称/
├── shaders/
│   ├── composite.fsh
│   ├── composite.vsh
│   ├── final.fsh
│   ├── final.vsh
│   ├── gbuffers_terrain.fsh
│   ├── gbuffers_terrain.vsh
│   └── 其他着色器文件...
└── shaders.properties
```

## 主要着色器文件类型
gbuffers阶段：处理基本几何体渲染，包括地形、实体、天空等。这里定义物体的基本外观和材质属性。
composite阶段：进行后处理效果，比如光照计算、阴影、反射、景深等高级视觉效果。
final阶段：最终的屏幕输出处理，通常用于色彩校正、色调映射等。


## 配置文件
shaders.properties文件用于定义shader的各种属性和设置选项：

```
# 基本信息
version=1.0
author=你的名字

# 渲染设置
shadowMapResolution=2048
shadowDistance=128.0

# 自定义选项
option.MOTION_BLUR=true
option.BLOOM=false
```


## 各阶段作用与影响范围

### gbuffers_basic.fsh（几何阶段）
作用：给场景物体渲染基础颜色/法线/深度等 G-Buffer。这里改颜色=改变“源素材”。
影响范围：仅世界几何（方块/实体）。不会直接覆盖 UI；但其输出会被后续阶段读取。如果后续阶段没正确采样，看到的就是这里的颜色被放大成整屏效果。
### composite.fsh（合成阶段）
作用：把前面的缓冲合成成一张“场景色”图（色调映射、SSR、水面等通常在这）。这里直通/调色会直接改变最终世界画面。
影响范围：世界画面为主。若写死红色，整屏会红（常见于我们之前的测试线）。
### final.fsh（最终阶段）
作用：对 composite 的结果做最后的全屏处理（色彩校正、UI 混合、FXAA 等）。
影响范围：最高优先级。这里若输出常量色，会覆盖世界与大多数 UI（你看到“UI 也变色”的典型表现）。


### block.properties
vBlockId = mc_Entity.x; 把编号放进去，片元阶段可读
通过映射让 mc_Entity.x 变为稳定值，比如 1000=water，1001=glass，避免不同版本原生ID不一致

### gbuffer_water gbuffer_translucent
你说得对：在你的运行环境里，gbuffers_water 这个程序确实会处理到“非水”的半透明几何（例如玻璃/玻璃板）。因此，一旦把非水的早退逻辑去掉，水里的上色就会同样作用到玻璃上。这正是我们现在在水的着色器里做 blockId 守卫和 return 的原因。

结论与做法

必须保留 gbuffers_water.fsh 的“非水早退”判断与 return（否则水的着色会影响玻璃）：
mc-shader/shaders/gbuffers_water.fsh:28
gbuffers_translucent 仍然有用，因为它负责“非水的所有半透明材质”（玻璃/玻璃板/冰/粘液等），并且我们已在这里统一用“仅纹理颜色”渲染，忽略 gl_Color 的绿色顶点色污染：
mc-shader/shaders/gbuffers_translucent.fsh:27
为什么会这样

Pass 名称不代表“只会处理某一类方块”。在某些版本/渲染管线下，部分半透明几何会被送进 gbuffers_water 程序。如果不在程序内部做 blockId 过滤，水的逻辑就会误作用到玻璃。

## 天空颜色（gbuffers_skybasic）

### 实现方案
完全弃用 vanilla 的逐顶点色 `gl_Color`，改为按视线高度 `h = dot(viewDir, upDir)` 解析地计算昼夜垂直渐变，再叠加由太阳高度驱动的多段日落色带（余烬→琥珀→珊瑚→品红→紫）。`viewDir` 由 `gl_FragCoord` + `gbufferProjectionInverse` 重建。

### 避免的坑
- **地平线硬缝**：MC 把天空拆成两块几何体渲染——上方穹顶 + 下方虚空/雾平面，两者 `gl_Color` 不同。直接用 `gl_Color` 当底色会原样继承这条几何缝。**解析地算渐变**即可彻底消除。
- 反推视线方向时 `clip.w` 为常数，所以 `normalize(vpos.xyz)` 不除以 w 也正确（除以同一个正标量不改变归一化方向）。
- 日落色带若各段 `smoothstep` 区间太窄、亮度跳变大，会出现"台阶"硬边；用更宽、相互重叠的区间 + 中间过渡色（珊瑚粉）桥接。最亮的暖色带要**跨过地平线**（区间起点为负），避免 h=0 处导数突变。

## 水面反射（gbuffers_water + composite）

### 实现方案（两段式）
1. **解析天空反射**（`gbuffers_water.fsh`）：把程序化天空沿反射光线方向采样，用 Schlick 菲涅尔（F0≈0.02）混合"青绿水色 ↔ 天空反射"。正视/近处通透、掠射/远处镜面。天空颜色复用 `gbuffers_skybasic` 同一套渐变。
2. **几何体 SSR**（`composite.fsh`）：反射屏幕上已渲染的树木/地形/实体。视空间反射光线步进 + 深度命中检测，命中采样 `colortex0`，按菲涅尔混合；未命中回退到天空反射。水面法线用世界 up（平静镜面）。

### 为什么反射必须在 composite 做
渲染顺序：gbuffers_terrain/entities → **gbuffers_water** → composite。在 gbuffers_water 阶段，后渲染的实体（甚至部分地形）还没进颜色缓冲，反射会缺东西。composite 在整个场景渲染完之后跑，`colortex0` 才是完整场景。

### 避免的坑（都踩过）
- **不要用颜色/蓝度判断水面**：极不可靠，尤其水面会反射橙色日落时直接失效。
- **不要依赖 colortex1 的清除值当遮罩**：实测 Iris 并没有把 colortex1 清成 alpha=0（结果整屏被判成水面、全屏品红）。MRT 写遮罩这条路放弃。
- **可靠的水面检测**：用 `depthtex0`（含半透明=水面深度）对比 `depthtex1`（不含半透明=水底深度），两者有明显视空间差值即为水面。**无需任何额外缓冲**。
- **排除云朵**：云也是半透明（`depthtex0<depthtex1`），但其背后是天空（`depthtex1≈1.0`）。要求"背后有不透明物"（`depthtex1<0.9999`）即可排除云。代价：看不到底的深水也会被排除。
- **SSR overshoot**：步长几何增长后，固定命中阈值太小会让光线"一步跨过树木判未命中"，导致完全没反射。**命中容差要随步长缩放**（`thick = max(floor, lastStep*1.6)`）。
- **水面自反射条带**：步进时若命中其他水面像素，会反射水面自身产生横向条带。步进时**跳过水面像素**（再次用 `isWaterAt`）。
- **调试手段**：composite 里加 `SSR_DEBUG` 开关，把检测到的水面涂品红，先确认遮罩通路再调反射数学——这一步直接定位了"colortex1 没清零"和"云误判"两个问题。
- SSR 固有局限：只能反射屏幕内可见的内容，屏幕外/被遮挡物体反射不出来，边缘需淡出。
## SSAO（composite → colortex3 → composite1）

### 实现方案
无法线 G-buffer 也能做 SSAO：从 `depthtex0` 重建视空间法线（每个轴取深度差更小的那侧邻居做差分，物体轮廓处不会出脏法线）。半球采样用黄金角螺旋 + interleaved gradient noise 逐像素旋转，全程不需要噪声贴图。原始 AO 写 `colortex3`，composite1 做 5x5 深度感知模糊后乘进场景色，之后才轮到 SSR（composite2），这样水面倒影里自带 AO。

### 避免的坑
- **AO 必须全屏每像素都写**：沿用"colortex 清除值不可靠"的教训，天空/超距像素显式写 1.0，不依赖清除值。
- **法线重建别直接用 dFdx/dFdy**：在物体边缘一侧的导数会跨物体取值，产生描边脏法线；改用"每轴取深度差小的一侧"的邻居差分。
- **采样点可能落到相机后方**（贴墙时），投影除 w 会得到垃圾 UV，需要 `sp.z >= 0 则跳过`。
- **原始 AO 噪声很大是正常的**，深度感知模糊是必需品；模糊权重用视空间 Z 差做门限，防止 AO 越过轮廓渗色（天空边缘发灰）。
- 原版平滑光照已经在顶点色里带了粗糙 AO，`AO_STRENGTH` 要克制，否则角落发黑过头。

## 空气透视 + god rays（composite3）

### 实现方案
- 空气透视：非天空像素按 `1-exp(-(dist/far)^2*density)` 向"该视线方向的程序化天空色"过渡。天空函数从 `gbuffers_skybasic.fsh` **复制**过来（OptiFine 不支持 #include，两处要手动同步）；地平线以下的 h 会被函数自然钳到地平线色带，正好就是想要的雾色，无需特判。
- god rays：屏幕空间径向步进，从像素朝太阳的屏幕投影点走 24 步，累计"天空可见性"并衰减；起步用 IGN 抖动消条带。不需要 shadow map。

### 避免的坑
- **太阳在相机背后时必须跳过**：`gbufferProjection * vec4(sunPosition,1)` 的 `w <= 0` 即背后，否则投影点会镜像到屏幕内产生假光柱。
- 太阳屏幕距离 ≥1 时 focus 权重为 0，提前跳过整个步进循环省开销。
- 水下要跳过空气透视（composite2 的水下色调已经处理），否则两层雾叠加。
- 验证顺序照旧：先开 `ATMO_DEBUG`/`AO_DEBUG` 看灰度通路，再看最终叠加。

## 云朵（gbuffers_clouds）

- pack 里原本没有 `gbuffers_clouds`，云走 fallback 程序；新建该文件即可完全接管云的着色，几何不动。
- 云的面法线**不要用顶点数据**，用 `dFdx/dFdy(viewPos)` 叉积重建（方块云全是平面四边形，绝对可靠），再翻转朝向眼睛——顺带解决背面渲染时法线反向的问题。
- "棉花糖感"的三要素：大 wrap 的漫反射（消掉硬明暗交界）+ 阴影面用粉蓝色而非灰色 + rim 透光边。对比度一定要低。
- 世界坐标锚定的噪声会随云漂移缓慢"流动"，可接受；若要完全钉在云上需要云的 UV/偏移 uniform，OptiFine 没有直接提供。

### 云朵圆角（屏幕空间覆盖率侵蚀，composite3）
- **踩坑：不要在 gbuffers_clouds 里靠采样 clouds.png 找云单元边界**。第一版假设"texcoord 在 clouds.png 空间、一纹素=12 格单元、采邻居纹素判断空气"，实测完全无效——Sodium 用自己的云渲染器，几何不带可用的云贴图 UV，采样永远返回"实心"，侵蚀静默变成空操作（无报错、云照常渲染，极难察觉）。凡是依赖"某 pass 的 UV 映射约定"的方案，都要先想想 Sodium 在不在场。
- 可靠做法：**后处理阶段按屏幕空间覆盖率侵蚀**。云掩码复用 SSR 那套深度对比（半透明 + 背后是天空 `depthtex1>=0.9999`），再加世界高度窗口（100~260）排除贴天空的玻璃/粒子。对云像素在随距离缩放的圆盘内打 16 个金角螺旋 tap 统计覆盖率：内部≈1、直边≈0.5、外凸直角≈0.25，把低覆盖率像素向解析天空色渐隐 → 尖角削得最多 = 圆角；阈值叠世界坐标噪声 = 绒毛边。
- 侵蚀是"把云画回天空"，所以**必须有解析天空函数可复用**（composite3 里本来就有），否则无从恢复被云覆盖的背景。
- 圆盘半径要按 `R * proj / (-viewZ * 2)` 换算成 UV 并设上限（贴脸飞近云时半径爆炸）。
- 云像素要**豁免空气透视雾**：云距离动辄数百米，雾会把云的着色洗掉 70% 以上（表现为云惨白无对比）。
- 局限：只能圆化"以天空为背景"的轮廓；云叠云、云压地形处的边仍是直角；被侵蚀的边缘会抹掉太阳/月亮贴图；水面 SSR 反射的是圆角前的云。

## 夜晚亮度与火把光晕（gbuffers 光照）

- **环境光不能只看天空 lightmap**：lightmap 的 sky 分量表示"能否看到天空"，深夜室外它仍是满值。若 `ambient = mix(min, max, envL)` 不乘太阳高度，深夜地面会保持白天亮度。修法：上端改为 `mix(nightAmbient, dayAmbient, day)`，其中 `day = clamp(sunUp)`。
- **火把增益指数决定"光晕感"**：`pow(torch, 0.35)` 半程仍有 ~0.86 强度，光不衰减 = 看不出发光；`pow(torch, 1.5)` 才有"中心亮、向外衰减"的光池。地面基础亮度越暗，这个衰减对比越明显——两者要一起调。
- 光照代码在 7 个 gbuffers 变体里各有一份（OptiFine 无 #include），改光照必须 grep 全部 `float ambient = mix` 逐一同步，漏一个就会出现某类方块（树叶/实体/地形）亮度不一致。

## 水面波浪（gbuffers_water + composite2）

- **只扰动法线，不动几何/透明度公式**：波浪 = 把喂给菲涅尔和反射的法线换成波动法线，`fres` 公式本身不变，"近处通透、掠射镜面"的菲涅尔行为自动保留，且太阳 glint、alpha 随波闪烁是免费副产品。
- **踩坑：几组方向正弦波叠加会干涉成规则格子/布纹**。第一版高度场用 4 组固定波矢的 `A*sin(dot(p,k)+ωt)` 相加，解析导数虽然精确省算力，但两组主波方向一交叉，整个水面就是明显的网格状驻波图案，非常假。修法：高度场改成**多八度 2D 值噪声，每个八度往不同方向漂移**（`wnoise2(p*freq + t*driftDir)`），永不重复成格子；梯度用中心差分（ε≈0.35），smoothstep 插值保证梯度连续。法线 `normalize(vec3(-g.x, 1, -g.y))`，世界坐标 `wp.xz` 锚定，图案不随镜头游动。
- **波浪强度必须随距离衰减**（`exp(-dist/40)` 之类）：远处水面一个像素跨多个波长，法线高频抖动 = 大面积闪烁噪点；衰减后远水回到平静镜面，掠射反射也更干净。
- 两个 pass 都要扰动（gbuffers 的天空反射 + composite2 的 SSR），且**必须用同一个波浪函数和参数**（复制同步），否则天空反射和几何反射的波纹对不上，水面看起来"两层皮"。
- SSR 的波动法线可能把掠射光线反射到水平线以下，光线往下步进会命中水底 → 倒影里出现水底碎块。判断世界空间 `rayDir.y < 0` 时回退平面镜法线。
- 只让**朝上的面**起波（`dot(vNormal, up)` 做权重），瀑布侧面保持顶点法线，否则侧流水面波纹方向错乱。
- `frameTimeCounter` 是 OptiFine/Iris 提供的秒计时 uniform，直接乘速度当相位即可（约 3600s 回绕一次，瞬间跳变肉眼几乎不可察）。
- **水下视角是独立分支，效果不会自动继承**：`isEyeInWater == 1` 走的是另一条代码路径，水上做的波浪/菲涅尔对它完全不生效，得单独实现。水下正确的物理是水→空气界面：`sin(空气) = 1.33 * sin(水)`，超过临界角（~48.6°）全反射成镜面，正上方是明亮的斯涅尔窗口；菲涅尔要用**空气侧折射角** `cosT` 代入 Schlick，角度趋近临界角时自然趋于 1。
- 水下"波光粼粼"的便宜做法：`refract(I, N, 1.33)` 把视线折射出水面，与太阳方向做 `pow(dot, 48)`——波浪法线逐像素扰动对齐度就是闪烁；TIR 时 refract 自动返回零向量，闪光在窗口外自然消失，无需特判。
